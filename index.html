🧠 The Theory Behind the Magic

1. Recurrence Relation Derivation

Merge Sort follows the Divide & Conquer paradigm. Here is how we get the formula:

  • Divide: Splitting the array into two halves takes constant time, O(1).
  • Conquer: We recursively sort the two halves. Since each half is size n/2, this takes 2T(n/2) time.
  • Combine: Merging the two sorted halves back together requires iterating through all elements, which takes linear time, O(n).

Total Time: T(n) = 2T(n/2) + O(n)

2. Complexity Calculations

Time Complexity: O(n log n)
Using the Master Theorem (or a Recursion Tree), the array is divided in half log(n) times. At each level of the tree, merging takes O(n) work. Therefore, O(n) work done log(n) times = O(n log n) for Best, Average, and Worst cases.

Space Complexity: O(n)
Unlike Quick Sort, Merge Sort is not an in-place sorting algorithm. During the Combine phase, we create temporary arrays (Left and Right) to hold the split elements before writing them back in sorted order. These temporary arrays require O(n) extra space.
